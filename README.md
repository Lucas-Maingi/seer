# Seer — KYC Document Verification, End to End

Seer is a production-shaped identity-document verification service: photograph a
Kenyan national ID or passport, and Seer localizes the document, rectifies it,
reads its fields, verifies the holder's face against a live selfie, and scores
the image for tampering — all on CPU, inside a measured latency budget.

It is built as **trust infrastructure**: every model in the pipeline is trained
on synthetic specimen documents generated by Seer itself, so the system is fully
reproducible without ever touching a real person's identity document.

## Why this is not a YOLO tutorial

Most "document detection" demos draw an axis-aligned box around an ID and stop.
Seer makes deliberately harder choices at every stage, because each one is the
choice a real KYC system has to make:

| Stage | Obvious approach | What Seer does instead |
|---|---|---|
| Data | Scrape real IDs (illegal) | **Synthetic specimen engine** — procedural Kenyan ID/passport renderer with per-pixel ground truth (corners, field polygons, text, tamper masks) and domain randomization |
| Localization | Bounding-box detector | **4-corner keypoint regression** with differentiable soft-argmax (DSNT) heatmaps → sub-pixel corners → **homography rectification** |
| OCR | Call Tesseract | **Custom CRNN + CTC** recognizer trained on our own synthetic field crops, plus **ICAO 9303 MRZ parsing with check-digit validation** |
| Face match | "Same person? looks similar" | **ArcFace embeddings**, cosine similarity with a threshold **calibrated to a target FMR** on a public benchmark |
| Tampering | Skip it | **Forensic fusion**: Error Level Analysis + noise-residual statistics + a CNN trained on synthetically forged documents |
| Deployment | `model.predict()` in Flask | **ONNX export → INT8 static quantization** with a calibration set, and a benchmark harness reporting p50/p95 CPU latency per stage |

## Pipeline

```
                 ┌────────────────────────────────────────────────────┐
                 │                     Seer API                       │
photo ──────────▶  1. localize   4-corner DSNT keypoints (ONNX INT8) │
                 │  2. rectify    DLT homography → canonical crop     │
                 │  3. read       CRNN field OCR + MRZ checksums      │
selfie ─────────▶  4. verify     ArcFace embed · cosine · calibrated │
                 │  5. inspect    ELA + residuals + tamper CNN        │
                 │  6. decide     field validity × face × forensics   │
                 └────────────────────────────────────────────────────┘
                                → verdict JSON + per-stage latency
```

## Data ethics, by construction

- **No real identity documents are used anywhere** — not for training, testing,
  or demos. The synthetic engine renders fictional personas (Faker) onto
  specimen layouts with faces that belong to no living person.
- Every generated document carries a baked-in **SPECIMEN** watermark, and the
  renderer makes no attempt to reproduce genuine security features (holograms,
  microprint, ghost images). It produces training data, not forgeries.
- Public research benchmarks (MIDV-2020 for localization, LFW for face
  verification) quantify the sim-to-real gap.

This mirrors how commercial KYC vendors actually work: real documents are
legally radioactive (Kenya Data Protection Act 2019, GDPR), so the synthetic
data pipeline *is* the product.

## Repository layout

```
seer/
  synth/       synthetic specimen engine: personas, faces, templates,
               scene compositing, domain randomization, tamper synthesis
  localize/    corner-keypoint model (DSNT), training, homography rectify
  ocr/         CRNN+CTC recognizer, field schema, MRZ (ICAO 9303) parser
  face/        ArcFace embedding, verification, threshold calibration

  runtime/     ONNX export, INT8 quantization, latency benchmark, pipeline
  api/         FastAPI service
web/           Next.js front end
scripts/       dataset generation / training / benchmark entry points
tests/         unit tests (checksums, geometry, codecs — the parts that lie)
```

## Quickstart

```bash
python -m venv .venv && . .venv/bin/activate   # or .venv\Scripts\activate
pip install -e ".[dev]"

# 1. generate a synthetic training set (documents + scenes + forgeries)
python -m seer.synth.generate --out data/synth --count 5000

# 2. train the pipeline stages
python -m seer.localize.train --data data/synth
python -m seer.ocr.train      --data data/synth
python -m seer.forensics.train --data data/synth

# 3. export + quantize + benchmark
python -m seer.runtime.export   --all
python -m seer.runtime.quantize --all --calib data/synth
python -m seer.runtime.bench    --report runs/latency.md

# 4. serve
uvicorn seer.api.main:app --reload
```

## Training the real thing

Training is split from serving by design: **generate anywhere, train on a
free cloud GPU, serve on a modest CPU.** The models are small (3M / 8M /
<1M parameters), so a free Colab T4 or Kaggle session trains all three in
an afternoon; the laptop only ever runs inference.

- **Cloud (recommended):**
  [![Open in Colab](https://colab.research.google.com/assets/colab-badge.svg)](https://colab.research.google.com/github/Lucas-Maingi/seer/blob/main/notebooks/train_colab.ipynb)
  — select a GPU runtime, run all cells, and download the ~tens-of-MB
  weights zip back to the serving machine.
- **Anywhere:** `bash scripts/train_all.sh` reproduces the whole sequence
  (dataset → three training runs → ONNX export → INT8 quantization →
  latency report). Tune with `COUNT=8000 EPOCHS_CORNER=30 ...`.

Back on the serving machine:

```bash
unzip seer_weights.zip -d weights/
python scripts/fetch_models.py     # YuNet + ArcFace for the face stage
python -m seer.runtime.bench       # the latency that counts: serving CPU
uvicorn seer.api.main:app          # + `npm run dev` in web/
```

Then evaluate the sim-to-real gap on MIDV-2020 (localization) and LFW
(face calibration via `python -m seer.face.calibrate`). Canonical renders
default to JPEG q95, keeping an 8k-sample dataset around 4 GB.
